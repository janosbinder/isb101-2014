UNIX Tutorial Six

Other useful UNIX commands  

quota

On multiuser computer users are allocated a certain amount of disk space on the file system for their personal files. On your personal computer or laptop no quota should be set at all. If you go over your quota, you are cannot write any more files.

To check your current quota on this computer and how much of it you have used, type

% quota -v

df

The df command reports on the space left on the file system. For example, to find out how much space is left on the fileserver, type

% df .

du

The du command outputs the number of kilobytes used by each subdirectory. Useful if you have gone over quota and you want to find out which directory has the most files. In your home-directory, type

% du

gzip

This compresses a file. For example, to zip tutorial_R1_001.fa, type

% gzip tutorial_R1_001.fa

This will zip the file and place it in a file called tutorial_R1_001.fa.gz

To unzip the file, use the gunzip command.

% gunzip tutorial_R1_001.fa.gz

tar

tar creates and manipulates streaming archive files. Archives can contain files and directories. For example, archive and zip the file list2 and the directory backups in your unixstuff directory

% tar cfz myArchive.tar.gz list2 backups

and unpack it in your backups directory:

tar -zxvf myArchive.tar.gz

file

file classifies the named files according to the type of data they contain, for example ascii (text), pictures, compressed data, etc.. To report on all files in your working directory, type

% file *

history

The shell keeps an ordered list of all the commands that you have entered. Each command is given a number according to the order it was entered.

% history (show command history list)

Exercise 6a

If you want to get all cp commands that you are used so far type

% history | grep cp <\p>
